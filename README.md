# SD Backup

Приложение для Linux: создание дампов SD-карт, сохранение образов с названием и обложкой, просмотр сохранённых образов и восстановление дампа на выбранную SD-карту.

## Возможности

- **Создание дампа** — выбор съёмного устройства (SD/USB), указание названия дампа и опциональной картинки-обложки. Образ сохраняется в папку (по умолчанию `~/SD-Backups`).
- **Список образов** — в главном окне отображаются карточки сохранённых образов (обложка, название, размер, дата).
- **Восстановление** — выбор сохранённого образа и целевой SD-карты; после подтверждения образ записывается на устройство (данные на нём будут перезаписаны).
- **Папка бэкапов** — можно указать другую папку для хранения образов через кнопку «Папка бэкапов...».

## Требования

- Python 3.10+
- PyQt6
- `dd`, `pkexec` (обычно есть в системах с PolicyKit)
- Права на чтение/запись блочных устройств (запрос пароля через pkexec при создании/восстановлении)

### Системные зависимости для GUI (Qt)

На Ubuntu/Debian для работы окна приложения нужна библиотека плагина xcb:

```bash
sudo apt install libxcb-cursor0   # обратите внимание: «cursor0» с нулём
```

Если после установки окно всё равно не открывается, попробуйте запуск с платформой Wayland:

```bash
QT_QPA_PLATFORM=wayland ./run.sh
```

## Установка

```bash
cd /path/to/backup_storage
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Запуск

**Важно:** запускать нужно Python из виртуального окружения (где установлен PyQt6).

Через скрипт (рекомендуется — не нужно активировать venv вручную):

```bash
./run.sh
```

Или вручную с активированным venv:

```bash
source .venv/bin/activate
python main.py
```

При первом создании дампа или восстановлении система запросит пароль администратора (через pkexec) для доступа к устройству.

## Структура проекта

- `main.py` — точка входа.
- `src/core/devices.py` — получение списка съёмных блок-устройств (lsblk).
- `src/core/dump.py` — создание и восстановление образов через `dd` с правами root (pkexec).
- `src/core/backups.py` — метаданные образов (имя, обложка, путь, дата) в `backups.json` в папке бэкапов.
- `src/gui/main_window.py` — главное окно с сеткой карточек.
- `src/gui/cards.py` — виджет карточки образа (обложка, название, кнопки).
- `src/gui/dialogs.py` — диалоги «Новый дамп», «Восстановление» и прогресс операций.

Образы сохраняются как файлы `.img`; обложки хранятся по выбранному пользователем пути (в метаданных сохраняется только путь к файлу обложки).

## Сборка AppImage

Чтобы получить один переносимый файл приложения с иконкой в меню:

1. Установите зависимости для иконки (опционально, для PNG в меню):  
   `sudo apt install librsvg2-bin`
2. Запустите сборку из корня проекта:

```bash
./build_appimage.sh
```

В корне появится файл `SD-Backup-x86_64.AppImage`. Сделайте его исполняемым (если нужно) и запускайте:

```bash
chmod +x SD-Backup-x86_64.AppImage
./SD-Backup-x86_64.AppImage
```

При первом запуске можно «интегрировать в систему» (создать пункт в меню с иконкой), если ваш рабочий стол это поддерживает. Скрипт при сборке скачивает `appimagetool` с GitHub (один раз).
